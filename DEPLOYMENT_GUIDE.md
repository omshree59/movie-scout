# MovieScout Deployment Guide

This guide covers how to deploy the frontend to Vercel and the backend to Railway.

## 1. Deploy Frontend to Vercel

Vercel is perfect for Vite/React applications.

### Steps:
1. Go to [Vercel](https://vercel.com/) and sign in with GitHub.
2. Click **Add New... > Project**.
3. Import your `movie-scout` GitHub repository.
4. **Configure the Project:**
   - **Framework Preset**: Vercel should auto-detect **Vite**.
   - **Root Directory**: Click "Edit" and change it to `frontend`.
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
5. **Environment Variables:**
   - Open your local `frontend/.env` file.
   - Copy each line and paste it into Vercel's Environment Variables section. (Paste the whole list in one box, and Vercel will auto-split them into VITE_FIREBASE_API_KEY, etc.)
   - *Important:* For your backend API URL, add a new variable:
     - Name: `VITE_API_BASE_URL`
     - Value: We will come back and set this to your Railway URL later. For now, you can leave it blank or put a placeholder.
6. Click **Deploy**. Vercel will build and launch your site.

---

## 2. Deploy Backend to Railway

Railway runs Python/FastAPI seamlessly using Docker or Nixpacks.

### Prepare the Backend First
Before jumping into Railway, ensure your backend has a `requirements.txt` and a `Procfile` (or Railway will try to guess). 

1. Create a `Procfile` in the `backend/` directory with this content:
   ```
   web: uvicorn app.main:app --host 0.0.0.0 --port $PORT
   ```
2. Make sure your `backend/requirements.txt` is fully updated.

### Steps on Railway:
1. Go to [Railway.app](https://railway.app/) and sign in with GitHub.
2. Click **New Project** > **Deploy from GitHub repo**.
3. Select your `movie-scout` repository.
4. Railway will analyze the repo and create a service.
5. **Configure the Service:**
   - Click on the newly created service.
   - Go to **Settings** > **Service Details** and set the **Root Directory** to `/backend`.
6. **Environment Variables:**
   - Go to the **Variables** tab.
   - Add your TMDB and Cloudinary variables:
     - `TMDB_API_KEY`: <your_tmdb_key>
     - `CLOUDINARY_CLOUD_NAME`: <your_cloud>
     - `CLOUDINARY_API_KEY`: <your_key>
     - `CLOUDINARY_API_SECRET`: <your_secret>
   - Add your Firebase Service Account JSON. Since Railway needs this file, the easiest way is to add a variable named `FIREBASE_CREDENTIALS_JSON` and paste the *entire contents* of your `firebase_credentials.json` directly into the value field. *(Note: you will need to tweak your `firebase_service.py` to read from this env var if it's currently hardcoded to look for a file).*
7. **Expose the API:**
   - Go to **Settings** > **Networking**.
   - Click **Generate Domain**. This will give you a public URL (e.g., `movie-scout-backend-production.up.railway.app`).

---

## 3. Link them Together

1. Copy the public domain URL generated by Railway.
2. Go back to your frontend project in **Vercel**.
3. Go to **Settings** > **Environment Variables**.
4. Update or Add `VITE_API_BASE_URL` and set its value to your Railway URL (e.g., `https://movie-scout-backend-production.up.railway.app/api`).
5. Trigger a new deployment in Vercel so the frontend builds with the new backend link.
